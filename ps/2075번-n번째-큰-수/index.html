2075번 N번째 큰 수

문제

https://www.acmicpc.net/problem/2075

문제에서는 다음과 같은 NxN 크기의 2차원 배열이 주어진다.

- - - - -
12 7 9 15 5
13 8 11 19 6
21 10 26 31 16
48 14 28 35 25
52 20 32 41 49

특징이 하나 있다면, 모든 수는 자신의 위에 있는 수보다 크다는 것이다. 우리는 여기서 N번째로 큰 수를 구하면 된다.

문제 자체는 그냥 이해하기 어렵지 않다. 간단하다.

문제는 이제 메모리에서 발생한다. 메모리가 무려 12Mb 다. int 정수형에 32비트가 들어가니까.. 대략 int 정수가 3000개 넘짓하는 크기가 들어간다. (정확히는 3072개가 들어갈 수 있다.)

이 문제에서의 N 한도는 1500 이기에, 주의해서 봐야 할 사항이다.

풀이과정

나는 처음 봤을 때 dp 인줄 알았다. 왜냐하면 저기 명시되어 있는 특징 때문이었다. 저 특징 때문에 처음 딱 보고

ㅇ? 위에 있는 수를 이용해야 하나?

싶었다. 그래서 막 [1500][2] 배열가지고 이리저리 머리를 굴려봤는데, 도저히 생각이 나질 않았다.

그래서 일단 내가 늘 하던대로, 딱 보고 직관적으로 생각나는 풀이를 한번 떠올려봤다. 그렇게 떠오른 건 딱 N 크기만큼의 우선순위 큐를 하나 준비해서, 닥치는대로 집어넣는거다. 처음 내가 적었던 코드는 이거다.

#include <iostream>
#include <queue>
using namespace std;
int n,a;
priority_queue<int> pq;
int main(){
    cin >> n;
    for (int i=0;i<n*n;i++){
        cin >> a;
        pq.push(-a);
        if (pq.size()>n) pq.pop();
    }
    cout << -pq.top();
    return 0;
}

이랬더니 시간초과가 덜컥 나버렸다. cin cout 이 느리다는 사실은 알고 있었기에 곧장 scanf printf 로 바꿔봤는데, 맞았다.

#include <stdio.h>
#include <queue>
using namespace std;
int n,a;
priority_queue<int> pq;
int main(){
    scanf("%d",&n);
    for (int i=0;i<n*n;i++){
        scanf("%d",&a);
        pq.push(-a);
        if (pq.size()>n) pq.pop();
    }
    printf("%d",-pq.top());
    return 0;
}

솔직히 이게 맞을거라곤 생각하진 못했는데 막상 맞으면 굉장히 기분이 좋다. 기대하지 않는게 좋은건가 싶기도 하다.
